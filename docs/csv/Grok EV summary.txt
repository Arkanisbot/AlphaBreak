Technical Specification: Auto-Detected Trendlines with Regime-Aware Scoring
Document Version: 1.0
Date: April 2026
Feature Owner: Nicholas Major (Founder)
Status: Draft → Ready for Engineering Review
1. Feature Overview
Objective
Provide fully automatic trendline detection on any chart (1m to monthly) that is more accurate and actionable than TrendSpider’s implementation by incorporating historical trend pattern matching and current market regime context.
Key Differentiators vs Competitors
* TrendSpider: Pure geometric detection.
* AlphaBreak: Geometric detection + historical analog scoring + regime weighting + confidence probability.
Success Metrics
* Detection accuracy: ≥85% agreement with professional manual trendlines on backtested data.
* User engagement: Average 3+ trendlines used per session.
* Performance: <800ms to render on a 6-month daily chart (H100 or equivalent).
2. Functional Requirements
Core Capabilities
* Automatically detect and draw bullish support and bearish resistance trendlines on any timeframe.
* Support multi-timeframe confluence (show the same trendline projected across 15m / 1H / 4H / Daily).
* Display confidence score (0–100%) for each trendline.
* Show historical analog strength: “This setup matches 87% of similar historical trend breaks in the current regime.”
* Allow one-click actions: “Add to watchlist”, “Create pre-trade plan”, “Set alert on break”.
User Controls
* Toggle: “Show only High-Confidence lines” (default: ≥70%).
* Filter by regime (Trending / Ranging / High-Vol / Low-Vol).
* Sensitivity slider: Conservative / Balanced / Aggressive (affects minimum swing point distance).
* Export trendlines as image or data (JSON/CSV).
Edge Cases
* Choppy / low-liquidity stocks → lower confidence or hide weak lines.
* Earnings gaps or news events → temporarily suppress or flag affected lines.
* Very short timeframes (1m/5m) → higher noise filtering.
3. Technical Approach & Architecture
High-Level Flow
1. Swing Point Detection — Use fractal or zigzag algorithm (with ATR-adaptive thresholds) to identify significant highs/lows.
2. Candidate Trendline Generation — Connect valid swing points using linear regression or least-squares fit. Filter using minimum touch points (minimum 3) and R² threshold.
3. Regime Classification — Feed current market data into your existing historical trend model to determine the prevailing regime (e.g., Bull Trending, Bear Ranging, Volatility Expansion).
4. Historical Analog Matching — Query your trend database for similar slope, duration, and touch patterns in the same regime. Compute similarity score (cosine similarity on normalized features).
5. Final Scoring — Confidence = (Geometric Quality × 0.4) + (Historical Win Rate × 0.4) + (Regime Alignment × 0.2).
6. Rendering — Draw lines with color gradient (green → red based on confidence) + tooltip showing probability of break / bounce.
Core Technologies
* Frontend: Lightweight Charts (or TradingView Lightweight) + custom overlay layer (React/TypeScript).
* Backend/ML: Python (FastAPI) + your existing LightGBM/XGBoost + LSTM components for time-series pattern matching.
* Database: Store pre-computed historical trend patterns in a vector database (e.g., Pinecone or pgvector) for fast similarity search.
* Real-time: WebSocket for live updates when new bars arrive.
Data Requirements
* OHLCV data (real-time + historical).
* Pre-computed regime labels from your model.
* Historical trend break outcomes (success/failure database).
4. Non-Functional Requirements
* Latency: <1 second for full chart analysis on standard user hardware.
* Scalability: Support 10,000 concurrent chart views.
* Accuracy target: Backtested against 5 years of S&P 500 components.
* Explainability: Every line must show “why” tooltip (e.g., “3 touches + 78% historical success in Trending regime”).
5. Implementation Phases
Phase 1 (2–3 weeks): Basic geometric auto-trendline detection (match or beat TrendSpider baseline).
Phase 2 (3–4 weeks): Integrate regime classification + confidence scoring.
Phase 3 (2 weeks): Historical analog matching + UI polish + alerts.
Phase 4: Multi-timeframe projection + brokerage integration hooks.
6. Testing Strategy
* Unit tests: Swing point detection accuracy.
* Integration tests: End-to-end chart rendering.
* Backtesting: Compare AlphaBreak auto-trendlines vs manual + vs TrendSpider on 100+ stocks.
* User testing: A/B test with 50 beta users measuring preference and trade idea generation.
